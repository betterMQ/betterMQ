Delivery model

How messages leave BetterMQ

Publish — one-off webhook (__direct).

Enqueue — FIFO per queue + routing key.

Groups — fan-out to multiple member webhooks.

Flows — rate / parallelism on publish paths.

Schedules — cron or interval direct webhooks.

DLQ — per-source dead letters after retries or hard HTTP errors.

Standalone: set password in Settings for your sk_local_… token.

D

Dead letter queue

GET

Failed deliveries land in a per-source DLQ — pick the source that matches how the message was sent.

/v1/dlq

Queue enqueue → jobs.__dlq (queue name + .__dlq)

Direct publish → __direct.__dlq (one-off POST /v1/publish)

Group fan-out → one DLQ per member (__group.{group}.{member}.__dlq)

Flow profiles only rate-limit delivery — they do not change which DLQ bucket is used.
